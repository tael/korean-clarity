# 영어 공간 은유 직역 탐지 설계

작성일: 2026-06-07
상태: 설계 승인 (구현 대기)
선행 작업: 2026-06-07-evaluation-philosophy-realignment.md (ai축 두 축 재정렬)

## 배경과 목적

AI가 쓴 한국어에는 영어의 공간 은유를 그대로 옮긴 표현이 자주 나온다. 두 갈래(branch),
층위(layer), 결(grain), 축(axis), 차원(dimension), 지점(point), 맥락(context) 같은 단어를
추상 개념에 붙여 "답은 두 갈래예요", "논의의 결이 다르다", "두 축으로 분석" 식으로 쓴다.
한국어 화자는 추상 대상에 이런 공간 은유를 이렇게까지 자주 쓰지 않는다. AI가 추상을
공간으로 환원하는 버릇이 글에 드러나는 것이다.

이 도구의 북극성은 "AI스러운 문장인가" 하나다. 번역 교정도, 국어 순화도 아니다. 따라서
이 탐지의 진단 근거도 두 축으로만 환원한다: (가) AI 전형성(AI가 추상에 공간 은유를 즐겨
씀) (나) 글 영향(구체 뜻이 흐려지고 문장이 겉멋만 듦). 메시지에 "영어 X의 직역" 같은
출신 낙인은 넣지 않는다.

## 핵심 긴장: 정착도가 제각각

이 단어들은 한국어 정착도가 단어마다 다르다. 일률 매칭하면 정상 한국어를 잘못 잡는
과탐이 난다.

- 맥락·차원: 거의 정착. "맥락을 보면", "다른 차원의 문제"는 정상 한국어다.
- 층위·결·갈래·지점·축: 추상 대상에 붙으면 번역체 색이 강하다.
- 같은 단어도 구체/추상에서 갈린다. "산이 두 갈래로"(정상) vs "답은 두 갈래"(AI 티),
  "나무결"(정상) vs "논의의 결"(AI 티), "회전축"(정상) vs "두 축으로"(AI 티).

이 도구는 정밀도를 재현율보다 우선한다. 정상 한국어를 잘못 잡으면 도구 신뢰가 무너지지만,
AI 티 몇 개를 놓치는 미탐은 감당할 수 있다. 아래 결정은 모두 이 우선순위를 따른다.

## 결정 사항

### 1. 범위와 분류

- 새 카테고리를 만들지 않고 기존 D2 사전을 확장한다.
- 대상 단어 5개: 층위·결·갈래·지점·축.
- 정착어 맥락·차원은 사전에 넣지 않는다(원천에서 과탐 차단).
- score.ts 가중치(D2: 7), CategoryCounts 타입, prompt.ts 카테고리 라벨은 건드리지 않는다.

### 2. positive 동반어 게이트

구체(정상) 용법과 추상(AI 티) 용법을 가르는 장치다. 기존 contextSafe는 "안전어가
윈도우에 있으면 통과"(negative)인데, 새 필드는 반대로 "추상 동반어가 윈도우에 있어야만
잡음"(positive)이다.

```ts
interface LexiconEntry {
  word: string;
  natural?: string;
  contextSafe: string[];        // 기존: 있으면 통과 (negative)
  requiresAbstract?: string[];  // 신설: 없으면 통과 (positive)
  message?: string;             // 신설: 항목별 두 축 메시지 (없으면 기본 템플릿)
  severity: Severity;
}
```

checkers.ts runLexiconChecks 루프에 한 줄 추가한다. 기존 negative 게이트 직후에 positive
게이트를 둔다.

```ts
const safe = entry.contextSafe.some((s) => window.includes(s));
if (safe) continue;
if (entry.requiresAbstract && !entry.requiresAbstract.some((a) => window.includes(a))) continue;
```

동반어는 선행/후행을 가리지 않고 같은 윈도우(앞뒤 25자) 안에 있으면 인정한다.
"답은 두 갈래"(선행)와 "두 축으로 분석"(후행)이 둘 다 자연스러운 AI 티이기 때문이다.

literal 용법(나무결·회전축·산갈래·도착 지점)은 추상 동반어가 없으니 게이트가 자연히
거른다. 별도 contextSafe 목록은 신설 항목에는 두지 않는다(positive 게이트로 충분).

씨앗 동반어 목록(구현 후 튜닝 전제):

| 단어 | requiresAbstract 씨앗 |
|---|---|
| 층위 | 의미·논의·분석·개념·사고·담론·해석 |
| 결 | 논의·문제·사안·감정·이야기·의미·주장 |
| 갈래 | 답·의견·입장·해석·논의·방향·가능성·경우 |
| 지점 | 논의·사고·분석·문제·고민 |
| 축 | 분석·논의·평가·사고·접근·기준·관점·가치 |

### 3. 빈도 게이트: 두지 않음

positive 게이트가 이미 정밀도를 책임지므로 minRepeat 없이 1회부터 발화한다. 사전 체커에
출현 카운트 로직을 새로 넣지 않는다(과한 기계). 추상 동반어가 곁에 있는 단일 출현은 그
자체로 충분히 AI 티다.

### 4. 지점의 한계: v1은 미탐 수용

positive 게이트의 약점이다. "이 지점에서"(= at this point)는 추상 동반어 없이도 AI 티인데,
동반어가 없어서 놓친다. v1은 이 미탐을 수용한다. 지점은 논의·사고·분석 같은 추상 동반어가
곁에 있을 때만 잡힌다.

### 5. 인접 정리: 기존 D2 기본 메시지 두 축화

사전 체커를 어차피 건드리므로, 기존 D2 기본 메시지(현재 checkers.ts의
"영어 단어 직역 의심. 맥락 확인 필요")의 출신 낙인을 이번에 함께 제거한다. 북극성이
보편이므로 D2 도메인 명사 항목도 두 축 메시지를 따른다.

신설 항목 메시지 예: "추상 개념에 공간 은유. AI가 추상을 공간으로 환원하는 버릇.
구체어로 풀면 가벼워짐". severity는 low.

natural 대안은 단어가 아니라 구절이라 suggestion에 담는다. 예: 갈래 → "둘로 나뉜다·두 가지".

## 추가(2026-06-07): 갈래 모델 역전

갈래는 위의 positive 동반어 게이트(추상 대상에 쓸 때만 탐지)가 맞지 않는다는 것이 드러났다.
축·층위·지점은 구체/추상이 진짜로 갈리지만, 갈래는 추상이든 구체든 일상 우리말에서 자연스럽게
쓰이지 않고 사실상 AI가 즐겨 쓰는 단어다(사용자 판정: "AI 클리셰"). 자연스러운 자리는 굳은 구체
표현 몇 개뿐이다(양 갈래 머리, 갈래길).

그래서 갈래만 모델을 뒤집었다.
- requiresAbstract 제거 → 기본적으로 탐지.
- contextSafe에 구체 굳은 표현(머리·머리카락·길·땋)만 넣어 면제.

결과: "답은 두 갈래", "두 갈래로 보면", "문제는 두 갈래로", "산이 두 갈래로"까지 모두 탐지하고,
"양 갈래 머리", "갈래길"만 통과한다. 자연 대체어는 "둘"·"두 가지". 다른 네 단어는 positive 게이트
유지.
