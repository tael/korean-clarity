# korean-clarity 설계 문서

작성일: 2026-05-10
상태: v0.1 착수

## 한 줄 정체성

"AI가 쓴 듯한 한국어"를 잡아내고 다듬는 공개 도구·지침 모음. 표면 클리셰가 아니라 **의미 전달의 명료성**이 본질.

## 왜 만드는가

AI가 한국어로 답할 때 어색함이 새는 이유는 단순히 클리셰가 많아서가 아니다. 클리셰가 의미 없는 자리에 의미 있는 척 들어앉아 본문을 덮는다. 한자어를 빼거나 외래어를 순화한다고 해결되지 않는다. **글이 의미를 운반하는가**가 진짜 문제.

시중 한국어 글쓰기 자료는 풍부하지만 "AI 한국어 냄새"를 정면으로 다룬 자료는 드물다. 이 프로젝트의 자리는 거기.

## 산출물 3종 (한 레포)

1. **지침 MD** — 의미 명료성 본질부터 시작해 AI 클리셰·번역체·어휘까지 챕터로 정리
2. **Claude 스킬 패키지** — `~/.claude/skills/`로 설치하는 두 스킬 (`korean-clarity`, `ai-cliche-guard`)
3. **웹 평가 도구** — 텍스트를 붙여넣으면 점수와 진단을 보여주는 정적 사이트 (Astro + React island)

## 핵심 분류 (6 카테고리)

표면(A·B·C·D·E)과 본질(F)을 분리한다.

| 카테고리 | 이름 | 본질 |
|---|---|---|
| A | 수사 클리셰 | "핵심은", "X가 아니라 Y", "결론적으로" — AI 특유 |
| B | 포맷 클리셰 | 콜론 제목, 글머리 남용, 강조 마크다운, 이모지 — AI 특유 |
| C | 구조 번역체 | 무생물 주어, 이중피동, 긴 관형절 — 영어 직역 |
| D-1 | 표현구 직역 | "위치하고 있다", "~에 대한", "~을 통해" |
| D-2 | 단어 직역 | "합성기", "처리기" — 사전 + 공기어 검사 |
| E | 격 어색함 | "~의" 연속, ~적·~성·~화 남발, 명사 뭉치 |
| F | **의미 명료성** | 추상도, 정보 밀도, 단언 강도, 핵심 동사 비율, 메타 발화 비율, 주어-서술어 거리 |

## 점수 체계

두 축 분리.

```
AI 냄새 지수 (0-100, 낮을수록 좋음)
  = 100 × (0.45·densityA + 0.30·densityB + 0.15·densityD2 + 0.10·densityE_meta)

명료성 지수 (0-100, 높을수록 좋음)
  = 100 - 100 × (0.30·abstractDensity + 0.20·infoDensityPenalty
              + 0.15·hedgeRatio + 0.15·formalVerbRatio
              + 0.10·metaRatio + 0.10·subjectVerbDistanceNorm)
```

가중치는 v0.1 출시 후 사람 평가 데이터로 조정.

## 분석기 동작

```
입력 → 정규화 → 형태소 분석(kiwi-wasm) → 문장 분할
      → 카테고리 검사기 7종 병렬 실행
      → 두 축 점수 + 카테고리별 밀도
      → 결과 패키징 (하이라이트 좌표, 위반 메시지, 교정 후보)
```

각 검사기는 같은 인터페이스: `RuleChecker.check(doc) → Violation[]`. 패턴 데이터(JSON·YAML)와 코드를 분리해 외부 기여(PR)를 받는다.

### F 카테고리 측정 (핵심)

- **추상도**: 한자 추상명사(~성·~화·~적·동작명사) 비율
- **정보 밀도**: 문장당 명사구 수 / 동사 수
- **단언 강도**: 단언형 vs 완충형 종결어미 비율
- **핵심 동사 비율**: 강한 고유어 동사 vs 형식 동사(하다·되다·이루어지다)
- **메타 발화 비율**: 메타 문장(분류는 A 검사기와 공유) / 전체 문장
- **주어-서술어 거리**: 형태소 분석에서 주어와 종결어미 사이 글자 수

### 입력 모드

- 본문 모드: 모든 카테고리 적용
- 표제·라벨 모드: A·D-2만 적용 (짧은 작업 항목·제목 검사용)

### 룰의 한계

룰 기반은 의미 부정합·맥락 어색·사실 오류는 잡지 못한다. v0.4부터 LLM 후속 진단(사용자 키)을 옵션으로 제공.

## 본질 점검 (실 사례)

세션 중 발견한 자기 풍자 사례. 내가 쓴 "분석기 야심 재설정"은 도구가 잡아야 할 표본이다.

- 1번: 학습 데이터의 영문 비즈니스 화법 ("redefine our ambition" → "야심 재정의")
- 2번: 비판에 대조 수사로 받는 관성 ("X가 아니라 Y")
- 3번: 추상명사로 헤더 다는 학습된 형식
- 4번: 세션 안 평행 압력 ("{명사}+{한자어 동사}" 반복)
- 5번: 출력 직전 자기 점검 부재

이 다섯이 도구의 표적. 4번은 정규식, 1·3번은 통계 지표, 2번은 응답 맥락 의존(LLM 후속).
